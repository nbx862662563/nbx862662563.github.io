闭包

发表于 2019-01-25 | 更新于 2019-02-16 | 分类于 JavaScript

概念：什么是闭包？

  • 闭包指的是能够获取另一个函数作用域中的变量的函数，也可以理解为一个作用域可以访问另一个作用域的变量

  • 进一步理解：因为js中只有全局作用域和函数作用域，在一个函数中想访问一个变量，会先在本函数作用域的内部查找，如果没有，则一层一层向外层作用域查找，但是不能反过来向内层（子作用域）查找。

  • 这时，如果外层函数就是需要使用内层的局部变量，就要使用闭包实现

闭包的用途

​ 在了解闭包的作用之前，我们先了解一下 Javascript 中的 GC 机制:

​ 在上述例子中，B 定义在 A 中，因此 B 依赖于 A ,而外部变量 C 又引用了 B , 所以A间接的被 C 引用。

​ 也就是说，A 不会被 GC 回收，会一直保存在内存中。为了证明我们的推理，上面的例子稍作改进：

​ count 是函数A 中的一个变量，它的值在函数B 中被改变，函数 B 每执行一次，count 的值就在原来的基础上累加，因此，函数A中的 count 变量会一直保存在内存中。

​ 当我们需要在模块中定义一些变量，并希望这些变量一直保存在内存中但又不会 “污染” 全局的变量时，就可以用闭包来定义这个模块。
