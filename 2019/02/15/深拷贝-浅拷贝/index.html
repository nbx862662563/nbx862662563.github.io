深拷贝/浅拷贝

发表于 2019-02-15 | 分类于 JavaScript

浅拷贝/深拷贝

遇到的问题：

​

​ 在修改数组b的时候，数组a的值也被修改了。

浅拷贝/深拷贝的概念：

​ 浅拷贝：新对象/数组 拷贝的只是原对象/数组的 引用；

​ 深拷贝：新对象/数组 创建新的，拷贝原对象/数组的 值；

​ 浅拷贝只复制指向某个对象的指针，而不复制对象本身，新旧对象还是共享同一块内存。但深拷贝会另外创造一个一模一样的对象，新对象跟原对象不共享内存，修改新对象不会改到原对象。

解决办法：

1.slice()方法

​ 数组.slice();//返回一个从原数组中截取一个片段组成新数组，不改变原数组；

​

​ slice(a,b)；参数1：截取数组起始索引； 参数2：截取数组终止索引；

​ 不带参数时，返回和原数组一样的新数组；

2.concat()方法

​ 数组.concat()；方法用于连接两个或多个数组。

​

​ 该方法不会改变现有的数组，而仅仅会返回被连接数组的一个副本。

3.Object.assign()方法

​ ES6的 Object.assign方法：对象深拷贝使用(将原对象的所有可枚举的属性和方法 复制到新的对象)

​

​ Object.assign方法适用于对象的深拷贝

4. JSON.parse(JSON.stringify(***)) 方法

​ 这是一个强大的方法！（通用）

​

5.ES6 扩展运算符

​ […obj] {…obj}

​ 总之，上述5种方法，一定能解决深拷贝/浅拷贝问题
